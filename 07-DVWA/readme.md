# 🛡️ Damn Vulnerable Web Application (DVWA)

## Overview

**Damn Vulnerable Web Application (DVWA)** is an intentionally vulnerable web application designed for learning and practicing web application security. During the CySecK Cybersecurity Finishing School, I used DVWA to understand how common web vulnerabilities arise, how attackers exploit them, and how secure coding practices can mitigate these risks.

The assessments were performed in a controlled local environment using **DVWA**, **XAMPP**, and **Burp Suite**, following ethical hacking principles.

---

# 🎯 Objectives

The primary objectives of this practical module were to:

- Understand common web application vulnerabilities.
- Perform hands-on security testing in a safe environment.
- Analyze insecure input validation and authentication mechanisms.
- Learn exploitation techniques and secure remediation practices.
- Strengthen practical web application security testing skills.

---

# 🧪 Vulnerability Assessments Performed

The following practical security assessments were successfully completed:

- SQL Injection (SQLi)
- Reflected Cross-Site Scripting (XSS)
- Stored Cross-Site Scripting (XSS)
- Command Injection
- File Inclusion
- Brute Force Authentication Testing

# 📋 Practical Methodology

Each assessment followed a structured approach:

1. Configure the local testing environment.
2. Identify the vulnerable module.
3. Observe normal application behaviour.
4. Manipulate user-controlled input.
5. Validate the vulnerability.
6. Analyze the security impact.
7. Document findings and mitigation recommendations.
