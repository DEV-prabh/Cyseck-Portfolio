# 🛡️ OWASP Top 10 Web Application Security

## Overview

This section documents my hands-on practice in identifying and exploiting common web application security vulnerabilities using **OWASP Juice Shop**, a deliberately vulnerable application designed for security training.

The objective was to understand how insecure implementations can be exploited and how these vulnerabilities can be mitigated using secure development practices.

---

## Objectives

- Understand the OWASP Top 10 security risks
- Perform practical web application security assessments
- Identify client-side and server-side security weaknesses
- Gain experience using industry-standard penetration testing tools
- Document findings using professional security reporting practices

---

## Environment

- **Target:** OWASP Juice Shop (Local Deployment)
- **Operating System:** Kali Linux
- **Primary Tool:** Burp Suite
- **Browser:** Chromium

## Key Learning

This practical exercise strengthened my understanding of how common web application vulnerabilities arise due to insecure authentication, inadequate server-side validation, and improper access control. It also reinforced the importance of secure coding practices and layered security controls in protecting modern web applications.
